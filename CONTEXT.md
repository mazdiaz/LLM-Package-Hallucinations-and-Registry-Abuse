## 1. Topic

**LLM Package Hallucinations and Registry Abuse** (a.k.a. *slopsquatting*)

Code-generating LLMs frequently emit references to software packages that do not exist in real registries. Attackers can register these hallucinated names with malicious code, creating a software supply-chain attack vector. The term "slopsquatting" was coined in April 2025 by Seth Larson (Python Software Foundation) and popularized by Andrew Nesbitt.

## 2. Pivot Locked

**Primary:** Pivot D — Registry Defense Benchmark
**Backup:** Pivot B — Agentic / MCP Hallucination (if D rejected)

### Pivot D Summary

Test whether PyPI and npm anti-abuse defenses actually catch hallucinated package names generated by current frontier LLMs. Build a lightweight classifier predicting which hallucinated names will evade defenses.

### Research Questions

- **RQ1:** How frequently do current PyPI/npm anti-abuse defenses (confusable-name rejection, quarantine, spam detection, squatting policy) catch hallucinated package names generated by state-of-the-art LLMs?
- **RQ2:** What features of a hallucinated package name predict evasion of registry defenses?
- **RQ3:** Can a lightweight classifier predict defense evasion with sufficient precision to serve as a pre-registration early-warning tool for registry operators?
- **RQ4:** How do hallucinated names distribute across the package-confusion taxonomy of Neupane et al. (2023), and which confusion categories correlate with defense evasion?

### Methodology (high-level)

1. Run N frontier LLMs (Claude, GPT-5, Gemini, DeepSeek, Llama, Mistral) on Spracklen-style prompt datasets.
2. Extract hallucinated package names.
3. Snapshot PyPI and npm registries (read-only ground truth).
4. Apply each registry's published anti-abuse rules to each hallucinated name; predict blocked/passes.
5. Engineer features per hallucinated name (lexical, similarity, model consensus, popularity asymmetry).
6. Train classifier predicting defense evasion.
7. Cross-reference Neupane et al. (2023) 13 confusion categories.
8. **Ethics:** Zero package registration; coordinated disclosure following Spracklen/Churilov precedent.

## 4. Critical Context — Field State

The peer-reviewed literature on LLM package hallucination is **extremely sparse**:

- **Only ONE peer-reviewed paper directly on this topic:** Spracklen et al. (USENIX Security 2025) "We Have a Package for You!" — 576,000 samples, 16 LLMs, 19.7% avg hallucination rate, 205,474 unique names.
- **Field is wide open** for follow-up peer-reviewed work.
- **arXiv preprints exist** (Churilov replication on 2026 frontier models; Krishna on Rust; Haque on Go; Twist on prompt variation) but are EXCLUDED from this proposal's source policy.

**Implication:** Any well-executed peer-reviewed follow-up to Spracklen is publishable. Differentiation is easy in peer-reviewed literature.
